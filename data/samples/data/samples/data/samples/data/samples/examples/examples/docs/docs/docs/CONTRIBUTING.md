# Contributing Guide

Thank you for improving climate-smart agriculture tooling.

## Principles
- Practical over theoretical
- Smallholder-first design
- Explainability over complexity
- Ethical and privacy-aware

## Contribution types
- Schema improvements
- New sample datasets
- Better baselines or evaluation methods
- Localization (languages, regions)
- Documentation clarity

## How to contribute
1. Fork the repository
2. Create a feature branch
3. Make small, focused changes
4. Open a pull request with context and evidence

## Commit style
- docs:
- feat:
- fix:
- chore:

## Quality expectations
- Keep artifacts usable with low connectivity
- Avoid unsafe or unverifiable agronomic advice
- Cite sources where applicable
